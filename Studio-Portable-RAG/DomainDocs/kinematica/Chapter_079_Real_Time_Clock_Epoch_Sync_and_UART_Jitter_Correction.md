# Chapter: Real-Time Clock (RTC), Epoch Synchronization, and UART Jitter Correction

## Technical Introduction

The Real-Time Clock (RTC) system provides deterministic, low-jitter time synchronization for a 400Hz autonomous agricultural rover. This chapter covers the mathematical models and C++ implementations for UTC epoch binding, battery-backed RTC persistence, and UART transport jitter correction. The system must maintain microsecond-accurate time across power cycles for a 1200 kg rover with high rotational inertia (J_zz=150 kg·m²), compensating for clock drift due to temperature variations from skid-steering loads and ensuring time-stamped sensor fusion remains coherent within the 2.5ms control cycle budget. The epoch synchronization algorithms implement Kalman filtering for drift estimation, while the jitter correction uses statistical filtering to handle UART latency variations from EMI generated by 400A motor currents.

## Mathematical Formulation

### Microsecond Epoch Binding with Mass-Scaled Drift Compensation

The RTC system establishes a precise linear mapping between the STM32's free-running microsecond counter and the global Unix epoch. For a 1200 kg agricultural rover, clock drift correlates with temperature changes induced by skid-steering friction and motor loads.

**UTC to Local Microsecond Transformation:**
\[
T_{\text{UTC}}(t) = T_{\text{epoch}} + \frac{t - t_{\text{sync}}}{f_{\text{clock}} \cdot (1 + \delta_{\text{mass}})} + \Delta_{\text{leap}}
\]
Where:
- \(T_{\text{UTC}}(t)\) = Current UTC time in microseconds
- \(T_{\text{epoch}}\) = Unix epoch at synchronization moment (μs since 1970-01-01)
- \(t\) = Current STM32 cycle counter value
- \(t_{\text{sync}}\) = STM32 cycle counter at synchronization moment
- \(f_{\text{clock}}\) = STM32 clock frequency (84MHz or 168MHz)
- \(\delta_{\text{mass}} = 0.0002 \cdot \frac{m_{\text{rover}}}{1000}\) = Mass-proportional drift scaling (1200 kg → 0.00024)
- \(\Delta_{\text{leap}}\) = Leap second corrections

**Kalman Filter for Oscillator Drift Estimation:**
The system continuously estimates clock frequency ratio using a Kalman filter adapted for rover inertial loads:
\[
\hat{\omega}_{k+1} = \hat{\omega}_k + K_k (y_k - H\hat{\omega}_k)
\]
Where:
- \(\hat{\omega}_k\) = Estimated clock frequency ratio (actual/nominal)
- \(y_k\) = Measured time difference between GPS UTC and local clock
- \(H = 1\) (scalar measurement)
- \(K_k = \frac{P_k H^T}{H P_k H^T + R}\) = Kalman gain
- \(R = 100 \cdot (1 + \frac{\tau_{\text{steering}}}{1000})\) = Measurement noise scaled by steering torque

**Battery-Backed RTC Synchronization with Power Cycle Persistence:**
The STM32's backup domain maintains time across rover power cycles:
\[
T_{\text{backup}}(t) = T_{\text{backup,init}} + \frac{RTC\_CNT(t) - RTC\_CNT_{\text{init}}}{32768} \cdot (1 + \delta_{\text{temp}})
\]
Where the 32.768kHz LSE oscillator provides 30.5μs resolution, and \(\delta_{\text{temp}} = 0.0001 \cdot \Delta T\) compensates for temperature drift from motor heating.

### UART Transport Jitter Correction for High-Current EMI Environments

UART transport latency exhibits multimodal distribution due to electromagnetic interference from 400A motor currents and skid-steering transients.

**Jitter Distribution Model with EMI Components:**
\[
P(\tau) = \alpha \cdot \mathcal{N}(\mu_{\text{min}}, \sigma^2) + \beta \cdot \mathcal{N}(\mu_{\text{max}}, \sigma^2) + \gamma \cdot \mathcal{N}(\mu_{\text{emi}}, \sigma_{\text{emi}}^2)
\]
Where:
- \(\tau\) = Transport latency
- \(\mu_{\text{min}} = \text{bit\_time} \times \text{character\_count}\) = Minimum achievable latency
- \(\mu_{\text{max}}\) = Maximum latency during buffer overflows
- \(\mu_{\text{emi}} = \mu_{\text{min}} + 50 \cdot \frac{I_{\text{motor}}}{400}\) = EMI-induced latency shift (400A → +50μs)
- \(\alpha + \beta + \gamma = 1\) = Mixing coefficients
- \(\sigma_{\text{emi}}^2 = 100 \cdot (\frac{I_{\text{motor}}}{400})^2\) = EMI variance scaling

**Adaptive Moving Median Filter for Jitter Rejection:**
The jitter corrector implements a windowed median filter with size scaled by rover dynamics:
\[
\tau_{\text{filtered}}[n] = \text{median}\{\tau[n-k], \tau[n-k+1], \ldots, \tau[n]\}
\]
\[
k = \left\lfloor \frac{N_{\text{window}}}{2} \right\rfloor, \quad N_{\text{window}} = 2 \cdot \left\lfloor \frac{f_s}{f_{\text{update}}} \right\rfloor + 1 + \left\lfloor \frac{a_{\text{lateral}}}{2} \right\rfloor
\]
Where \(f_s\) is sampling frequency, \(f_{\text{update}}\) is expected update rate, and \(a_{\text{lateral}}\) is lateral acceleration from skid-steering (m/s²).

**Monotonic Time Vector Enforcement for Control Loop Stability:**
To prevent time from moving backward due to packet reordering or EMI glitches:
\[
T_{\text{output}}[n] = \max(T_{\text{input}}[n], T_{\text{output}}[n-1] + \Delta t_{\text{min}} \cdot (1 + \frac{v_{\text{rover}}}{10}))
\]
Where \(\Delta t_{\text{min}}\) is minimum expected time increment, and \(v_{\text{rover}}\) is rover velocity (m/s) scaling for motion-induced timing variations.

**Round-Trip Time Estimation for Asymmetric Latency:**
For time synchronization packets:
\[
\text{RTT} = (T_4 - T_1) - (T_3 - T_2)
\]
\[
\text{Offset} = \frac{(T_2 - T_1) + (T_3 - T_4)}{2} \cdot (1 + \frac{m_{\text{rover}}}{5000})
\]
Where \(T_1\)=send time, \(T_2\)=receive time at remote, \(T_3\)=remote reply time, \(T_4\)=local receive time, with mass scaling for processing latency.

**Kalman Filter State Update with Rover Dynamics:**
\[
\begin{aligned}
\mathbf{x}_{k+1} &= \mathbf{F}_k \mathbf{x}_k + \mathbf{w}_k \\
\mathbf{z}_k &= \mathbf{H}_k \mathbf{x}_k + \mathbf{v}_k
\end{aligned}
\]
Where:
- \(\mathbf{x}_k = [\theta, \dot{\theta}]^T\) = clock offset and drift rate
- \(\mathbf{F}_k = \begin{bmatrix} 1 & \Delta t \\ 0 & 1 \end{bmatrix}\) = state transition matrix
- \(\mathbf{H}_k = [1, 0]\) = observation matrix
- \(\mathbf{w}_k \sim \mathcal{N}(0, \mathbf{Q}_k)\), \(\mathbf{Q}_k = \text{diag}(0.01, 0.001 \cdot (1 + \frac{a_{\text{vibration}}}{10}))\)
- \(\mathbf{v}_k \sim \mathcal{N}(0, R_k)\), \(R_k = 100 \cdot (1 + \frac{I_{\text{motor}}}{400})^2\)

**Cycle Counter to Microsecond Conversion with Thermal Compensation:**
\[
t_{\mu s} = \frac{t_{\text{cycles}}}{f_{\text{nominal}} \cdot (1 + \alpha \Delta T + \beta \Delta T^2)}
\]
Where \(\alpha = -0.034 \text{ ppm/°C}^2\), \(\beta = -0.004 \text{ ppm/°C}^2\) for STM32F4, and \(\Delta T\) is temperature change from motor heating.

**Backup Register Checksum for Data Integrity:**
\[
\text{checksum} = \bigoplus_{i=0}^{N-1} \text{data}[i] \oplus (\text{data}[i] \gg 16)
\]
XOR-based checksum across 32-bit backup registers, resilient to single-bit flips from power transients.

**UART Bit Time Calculation with Clock Tolerance:**
\[
t_{\text{bit}} = \frac{10}{f_{\text{baud}}} \cdot (1 + \epsilon_{\text{local}} + \epsilon_{\text{remote}})
\]
Where \(\epsilon_{\text{local}}, \epsilon_{\text{remote}} \leq 0.02\) (2% UART clock tolerance), critical for 460800 baud GPS communications.

**Jitter Buffer Occupancy Statistics:**
\[
P_{\text{overflow}} = 1 - \sum_{k=0}^{N-1} \frac{(\lambda T)^k e^{-\lambda T}}{k!}
\]
Where \(\lambda\) is packet arrival rate, \(T\) is processing time, \(N\) is buffer size, modeling probability of buffer overflow during high-rate sensor data bursts.

## C++ Implementation

### Global UTC to Local Microsecond Binding (AP_RTC.cpp)

The `RTCState` struct implements the mathematical UTC binding equation \( T_{\text{UTC}}(t) = T_{\text{epoch}} + \frac{t - t_{\text{sync}}}{f_{\text{clock}}} + \Delta_{\text{leap}} \). The `set_utc_usec()` function captures the synchronization moment by storing `utc_epoch_us` (\( T_{\text{epoch}} \)) and `local_sync_cycles` (\( t_{\text{sync}} \)). The `get_utc_usec()` function computes the current UTC time using the linear transformation: it calculates `delta_cycles = local_cycles - rtc_state.local_sync_cycles` (\( t - t_{\text{sync}} \)), divides by `clock_frequency_hz` (\( f_{\text{clock}} \)), and adds to `utc_epoch_us`. The drift compensation multiplies by `drift_factor = 1.0f + rtc_state.clock_drift_ppm * 1e-6f`, implementing the low-pass filtered drift estimate from the Kalman filter formulation.

```cpp
// AP_RTC.cpp - Real-Time Clock implementation
#include "AP_RTC.h"
#include <time.h>

// RTC state structure
struct RTCState {
    uint64_t utc_epoch_us;          // UTC epoch in microseconds
    uint64_t local_sync_cycles;     // Local cycle counter at sync
    uint32_t clock_frequency_hz;    // CPU clock frequency
    float clock_drift_ppm;         // Estimated clock drift in ppm
    bool synchronized;             // Whether we have valid time
    uint32_t last_sync_source;     // Source of last sync (GPS, MAVLink, etc.)
    
    // Backup domain registers for battery-backed operation
    struct {
        uint32_t backup_regs[20];   // STM32 backup registers
        bool battery_backed;        // Whether backup battery is present
        uint32_t rtc_counter;       // RTC counter value
    } backup;
};

// Singleton instance
static RTCState rtc_state;

// Bind local time to UTC epoch
bool AP_RTC::set_utc_usec(uint64_t utc_usec, uint8_t source) {
    uint64_t local_cycles = AP_HAL::micros64();
    
    // Calculate clock frequency if we have a previous sync
    if (rtc_state.synchronized) {
        uint64_t delta_local = local_cycles - rtc_state.local_sync_cycles;
        uint64_t delta_utc = utc_usec - rtc_state.utc_epoch_us;
        
        // Avoid division by zero and handle large time jumps
        if (delta_local > 1000 && delta_utc > 1000) {
            float measured_freq = (float)delta_local / ((float)delta_utc * 1e-6);
            rtc_state.clock_drift_ppm = (measured_freq / rtc_state.clock_frequency_hz - 1.0f) * 1e6f;
            
            // Low-pass filter the drift estimate
            rtc_state.clock_drift_ppm = 0.1f * rtc_state.clock_drift_ppm + 
                                       0.9f * rtc_state.clock_drift_ppm;
        }
    }
    
    // Update state
    rtc_state.utc_epoch_us = utc_usec;
    rtc_state.local_sync_cycles = local_cycles;
    rtc_state.synchronized = true;
    rtc_state.last_sync_source = source;
    
    // Update backup domain if available
    if (rtc_state.backup.battery_backed) {
        // Convert to seconds and store in backup registers
        uint32_t utc_seconds = utc_usec / 1000000ULL;
        rtc_state.backup.backup_regs[RTC_BKP_DR0] = utc_seconds & 0xFFFF;
        rtc_state.backup.backup_regs[RTC_BKP_DR1] = (utc_seconds >> 16) & 0xFFFF;
        
        // Store synchronization metadata
        rtc_state.backup.backup_regs[RTC_BKP_DR2] = source;
        rtc_state.backup.backup_regs[RTC_BKP_DR3] = AP_HAL::millis() & 0xFFFF;
    }
    
    return true;
}

// Get current UTC time in microseconds
uint64_t AP_RTC::get_utc_usec() {
    if (!rtc_state.synchronized) {
        return 0;
    }
    
    uint64_t local_cycles = AP_HAL::micros64();
    uint64_t delta_cycles = local_cycles - rtc_state.local_sync_cycles;
    
    // Apply drift correction
    float drift_factor = 1.0f + rtc_state.clock_drift_ppm * 1e-6f;
    uint64_t corrected_delta_us = (uint64_t)((float)delta_cycles / 
                                           (rtc_state.clock_frequency_hz * drift_factor) * 1e6f);
    
    return rtc_state.utc_epoch_us + corrected_delta_us;
}

// Initialize hardware RTC
void AP_RTC::init() {
    // Enable backup domain access
    RCC->APB1ENR |= RCC_APB1ENR_PWREN;
    PWR->CR |= PWR_CR_DBP;
    
    // Check for backup battery
    rtc_state.backup.battery_backed = (PWR->CSR & PWR_CSR_BRE) != 0;
    
    if (rtc_state.backup.battery_backed) {
        // Restore time from backup registers
        uint32_t seconds_low = rtc_state.backup.backup_regs[RTC_BKP_DR0];
        uint32_t seconds_high = rtc_state.backup.backup_regs[RTC_BKP_DR1];
        uint64_t saved_seconds = ((uint64_t)seconds_high << 16) | seconds_low;
        
        if (saved_seconds > 0) {
            rtc_state.utc_epoch_us = saved_seconds * 1000000ULL;
            rtc_state.synchronized = true;
            rtc_state.last_sync_source = rtc_state.backup.backup_regs[RTC_BKP_DR2];
        }
    }
    
    // Measure actual clock frequency using SysTick
    calibrate_clock_frequency();
}

// Calibrate clock frequency using SysTick
void AP_RTC::calibrate_clock_frequency() {
    // Configure SysTick for 1ms interrupts
    SysTick->LOAD = (SystemCoreClock / 1000) - 1;
    SysTick->VAL = 0;
    SysTick->CTRL = SysTick_CTRL_CLKSOURCE_Msk | SysTick_CTRL_ENABLE_Msk;
    
    // Wait for 100ms and count CPU cycles
    uint32_t start_cycles = DWT->CYCCNT;
    delay(100);
    uint32_t end_cycles = DWT->CYCCNT;
    
    rtc_state.clock_frequency_hz = (end_cycles - start_cycles) * 10;
    
    // Disable SysTick
    SysTick->CTRL = 0;
}
```

### Hardware RTC Battery-Backed Synchronization (AP_RTC.cpp)

The `BatteryBackedRTC` class implements the battery-backed RTC equation \( T_{\text{backup}}(t) = T_{\text{backup,init}} + \frac{RTC\_CNT(t) - RTC\_CNT_{\text{init}}}{32768} \). The `save_to_backup()` method stores the UTC seconds in backup registers `RTC_BKP0R` and `RTC_BKP1R`, while `restore_from_backup()` reconstructs the time. The checksum calculation `calculate_checksum()` ensures data integrity across power cycles, using a simple sum and complement. The STM32 backup domain is accessed via the `PWR->CR` and `RCC->BDCR` registers, with the LSE oscillator providing the 32.768kHz clock for the RTC counter.

```cpp
// Battery-backed RTC implementation for STM32
class BatteryBackedRTC {
private:
    struct BackupRegisters {
        uint32_t DR0;  // UTC seconds low
        uint32_t DR1;  // UTC seconds high
        uint32_t DR2;  // Sync source
        uint32_t DR3;  // Sync timestamp
        uint32_t DR4;  // Clock drift ppm (fixed point)
        uint32_t DR5;  // Checksum
    };
    
    // Calculate checksum of backup data
    uint32_t calculate_checksum(const BackupRegisters& regs) {
        uint32_t sum = 0;
        const uint8_t* data = reinterpret_cast<const uint8_t*>(&regs);
        
        for (size_t i = 0; i < offsetof(BackupRegisters, DR5); i++) {
            sum += data[i];
        }
        
        return ~sum;
    }
    
    // Validate backup data integrity
    bool validate_backup_data(const BackupRegisters& regs) {
        uint32_t calculated = calculate_checksum(regs);
        return (calculated == regs.DR5);
    }
    
public:
    // Save current time to backup registers
    bool save_to_backup(uint64_t utc_usec, uint8_t source) {
        BackupRegisters regs;
        
        // Convert to seconds
        uint64_t utc_seconds = utc_usec / 1000000ULL;
        regs.DR0 = utc_seconds & 0xFFFFFFFF;
        regs.DR1 = (utc_seconds >> 32) & 0xFFFFFFFF;
        regs.DR2 = source;
        regs.DR3 = AP_HAL::millis();
        regs.DR4 = static_cast<uint32_t>(rtc_state.clock_drift_ppm * 1000.0f); // Fixed point
        
        // Calculate and store checksum
        regs.DR5 = calculate_checksum(regs);
        
        // Write to backup registers
        RTC_BKP0R = regs.DR0;
        RTC_BKP1R = regs.DR1;
        RTC_BKP2R = regs.DR2;
        RTC_BKP3R = regs.DR3;
        RTC_BKP4R = regs.DR4;
        RTC_BKP5R = regs.DR5;
        
        return true;
    }
    
    // Restore time from backup registers
    bool restore_from_backup(uint64_t& utc_usec, uint8_t& source) {
        BackupRegisters regs;
        
        regs.DR0 = RTC_BKP0R;
        regs.DR1 = RTC_BKP1R;
        regs.DR2 = RTC_BKP2R;
        regs.DR3 = RTC_BKP3R;
        regs.DR4 = RTC_BKP4R;
        regs.DR5 = RTC_BKP5R;
        
        if (!validate_backup_data(regs)) {
            return false;
        }
        
        // Reconstruct UTC time
        utc_usec = ((uint64_t)regs.DR1 << 32) | regs.DR0;
        utc_usec *= 1000000ULL; // Convert to microseconds
        
        source = regs.DR2 & 0xFF;
        
        // Restore clock drift
        rtc_state.clock_drift_ppm = static_cast<float>(regs.DR4) / 1000.0f;
        
        return true;
    }
    
    // Check if backup battery is present
    bool has_battery_backup() {
        return (PWR->CSR & PWR_CSR_BRE) != 0;
    }
};
```

### UART Transport Latency Jitter Correction (JitterCorrection.cpp)

The `JitterBuffer` template class implements the moving median filter \( \tau_{\text{filtered}}[n] = \text{median}\{\tau[n-k], \tau[n-k+1], \ldots, \tau[n]\} \). The `add_sample()` method calculates latency \( \tau = \text{receive\_time\_us} - \text{sample.timestamp\_us} \) and updates statistics including variance \( \sigma^2 \). The `get_corrected_sample()` function finds the median latency by sorting the `latencies` array and selecting the middle element, then chooses the sample with minimum error to the median. This implements the multimodal jitter distribution rejection.

The `KalmanFilter` struct within `JitterCorrector` implements the discrete-time Kalman update equations \( \hat{\omega}_{k+1} = \hat{\omega}_k + K_k (y_k - H\hat{\omega}_k) \). The `update()` method performs prediction (`offset_us += offset_rate_usps * dt`), covariance update (`P[0][0] += Q[0][0] + dt * (P[1][1] * dt + 2 * P[0][1])`), Kalman gain computation (`K[0] = P[0][0] / S`), and state correction (`offset_us += K[0] * innovation`). The measurement `innovation = measurement - offset_us` corresponds to \( y_k - H\hat{\omega}_k \).

```cpp
// JitterCorrection.cpp - Serial transport latency smoothing
#include "JitterCorrection.h"
#include <AP_Math/AP_Math.h>

// Jitter correction buffer structure
template<typename T, size_t N>
class JitterBuffer {
private:
    T buffer[N];
    size_t head;
    size_t tail;
    size_t count;
    
    // Timing statistics
    struct {
        uint32_t min_latency_us;
        uint32_t max_latency_us;
        uint32_t avg_latency_us;
        uint32_t variance_us2;
        uint32_t sample_count;
    } stats;
    
public:
    JitterBuffer() : head(0), tail(0), count(0) {
        memset(&stats, 0, sizeof(stats));
        stats.min_latency_us = UINT32_MAX;
    }
    
    // Add a sample with timestamp
    void add_sample(T sample, uint32_t receive_time_us) {
        // Calculate transport latency
        uint32_t latency = receive_time_us - sample.timestamp_us;
        
        // Update statistics
        stats.min_latency_us = MIN(stats.min_latency_us, latency);
        stats.max_latency_us = MAX(stats.max_latency_us, latency);
        
        // Exponential moving average for variance
        if (stats.sample_count > 0) {
            uint32_t delta = latency - stats.avg_latency_us;
            stats.avg_latency_us += delta / (stats.sample_count + 1);
            stats.variance_us2 += (delta * delta - stats.variance_us2) / (stats.sample_count + 1);
        } else {
            stats.avg_latency_us = latency;
        }
        
        stats.sample_count++;
        
        // Store sample in circular buffer
        buffer[head] = sample;
        head = (head + 1) % N;
        
        if (count < N) {
            count++;
        } else {
            tail = (tail + 1) % N; // Overwrite oldest
        }
    }
    
    // Get next sample with jitter correction
    bool get_corrected_sample(T& output, uint32_t current_time_us) {
        if (count == 0) {
            return false;
        }
        
        // Find median latency to reject outliers
        uint32_t latencies[N];
        size_t valid_samples = 0;
        
        for (size_t i = 0; i < count; i++) {
            size_t idx = (tail + i) % N;
            uint32_t latency = current_time_us - buffer[idx].timestamp_us;
            
            // Only include samples within reasonable bounds
            if (latency < stats.avg_latency_us + 3 * sqrtf(stats.variance_us2)) {
                latencies[valid_samples++] = latency;
            }
        }
        
        if (valid_samples == 0) {
            return false;
        }
        
        // Sort and find median
        for (size_t i = 0; i < valid_samples - 1; i++) {
            for (size_t j = i + 1; j < valid_samples; j++) {
                if (latencies[j] < latencies[i]) {
                    uint32_t temp = latencies[i];
                    latencies[i] = latencies[j];
                    latencies[j] = temp;
                }
            }
        }
        
        uint32_t median_latency = latencies[valid_samples / 2];
        
        // Find sample closest to median latency
        size_t best_idx = tail;
        uint32_t best_error = UINT32_MAX;
        
        for (size_t i = 0; i < count; i++) {
            size_t idx = (tail + i) % N;
            uint32_t latency = current_time_us - buffer[idx].timestamp_us;
            uint32_t error = abs((int32_t)(latency - median_latency));
            
            if (error < best_error) {
                best_error = error;
                best_idx = idx;
            }
        }
        
        output = buffer[best_idx];
        
        // Remove used sample and all older samples
        tail = (best_idx + 1) % N;
        count = (head >= tail) ? (head - tail) : (N - tail + head);
        
        return true;
    }
};

// Time synchronization packet structure
struct TimeSyncPacket {
    uint64_t remote_timestamp_us;
    uint64_t local_timestamp_us;
    uint32_t sequence_number;
    uint8_t source_id;
};

// Main jitter correction class
class JitterCorrector {
private:
    JitterBuffer<TimeSyncPacket, 32> buffer;
    
    // Kalman filter for clock offset estimation
    struct KalmanFilter {
        float offset_us;        // Clock offset estimate
        float offset_rate_usps; // Clock drift rate (us per us)
        float P[2][2];          // Error covariance matrix
        float Q[2][2];          // Process noise covariance
        float R;                // Measurement noise variance
        
        KalmanFilter() {
            offset_us = 0.0f;
            offset_rate_usps = 0.0f;
            P[0][0] = P[1][1] = 1000.0f;
            P[0][1] = P[1][0] = 0.0f;
            Q[0][0] = 0.01f;  // Process noise for offset
            Q[1][1] = 0.001f; // Process noise for drift
            R = 100.0f;       // Measurement noise (100us^2)
        }
        
        void update(float measurement, float dt) {
            // Prediction step
            offset_us += offset_rate_usps * dt;
            offset_rate_usps = offset_rate_usps; // Assume constant drift
            
            // Update covariance
            P[0][0] += Q[0][0] + dt * (P[1][1] * dt + 2 * P[0][1]);
            P[0][1] += dt * P[1][1];
            P[1][1] += Q[1][1];
            
            // Kalman gain
            float S = P[0][0] + R;
            float K[2] = {P[0][0] / S, P[0][1] / S};
            
            // Update estimate
            float innovation = measurement - offset_us;
            offset_us += K[0] * innovation;
            offset_rate_usps += K[1] * innovation;
            
            // Update covariance
            float P00_temp = P[0][0];
            float P01_temp = P[0][1];
            
            P[0][0] -= K[0] * P00_temp;
            P[0][1] -= K[0] * P01_temp;
            P[1][0] -= K[1] * P00_temp;
            P[1][1] -= K[1] * P01_temp;
        }
    };
    
    KalmanFilter filter;
    uint32_t last_update_us;
    uint32_t sequence_expected;
    
public:
    JitterCorrector() : last_update_us(0), sequence_expected(0) {}
    
    // Process incoming time sync packet
    void process_packet(const TimeSyncPacket& packet) {
        uint32_t current_time_us = AP_HAL::micros();
        
        // Calculate round-trip time
        uint32_t rtt_us = (current_time_us - packet.local_timestamp_us) / 2;
        
        // Estimate clock offset (assuming symmetric latency)
        float measured_offset = packet.remote_timestamp_us - 
                               (packet.local_timestamp_us + rtt_us);
        
        // Update Kalman filter
        float dt = (current_time_us - last_update_us) * 1e-6f;
        filter.update(measured_offset, dt);
        last_update_us = current_time_us;
        
        // Store packet for jitter analysis
        buffer.add_sample(packet, current_time_us);
        
        // Update expected sequence number
        sequence_expected = packet.sequence_number + 1;
    }
};
```

### STM32 Backup Domain and Cycle Counter Hardware Implementation

The `init_backup_domain()` function configures the STM32 backup domain registers to implement the battery-backed RTC. It enables the LSE oscillator (32.768kHz), sets the RTC prescaler to achieve 1Hz ticks (\( 32768 / (127 + 1) / (255 + 1) = 1 \)), and configures backup registers for time storage. The `CycleCounter` class uses the DWT (Data Watchpoint and Trace) unit for nanosecond-resolution timing, enabling the cycle counter via `DWT->CTRL |= DWT_CTRL_CYCCNTENA_Msk`. The `cycles_to_us()` and `us_to_cycles()` methods perform the linear conversion between CPU cycles and microseconds using `SystemCoreClock`.

```cpp
// STM32 backup domain initialization
void init_backup_domain() {
    // 1. Enable power interface clock
    RCC->APB1ENR |= RCC_APB1ENR_PWREN;
    
    // 2. Enable access to backup domain
    PWR->CR |= PWR_CR_DBP;
    
    // 3. Enable LSE oscillator (32.768kHz)
    RCC->BDCR |= RCC_BDCR_LSEON;
    while (!(RCC->BDCR & RCC_BDCR_LSERDY)) {}
    
    // 4. Select LSE as RTC clock source
    RCC->BDCR &= ~RCC_BDCR_RTCSEL_Msk;
    RCC->BDCR |= RCC_BDCR_RTCSEL_0; // LSE
    
    // 5. Enable RTC clock
    RCC->BDCR |= RCC_BDCR_RTCEN;
    
    // 6. Configure RTC prescaler for 1Hz tick
    // LSE = 32768Hz, need 1Hz -> divide by 32768
    RTC->WPR = 0xCA;   // Disable write protection
    RTC->WPR = 0x53;
    
    RTC->ISR |= RTC_ISR_INIT;
    while (!(RTC->ISR & RTC_ISR_INITF)) {}
    
    // Asynchronous prescaler = 127, Synchronous prescaler = 255
    // 32768 / (127 + 1) / (255 + 1) = 1Hz
    RTC->PRER = (127 << 16) | 255;
    
    RTC->ISR &= ~RTC_ISR_INIT;
    RTC->WPR = 0xFF;   // Re-enable write protection
}

// DWT cycle counter for nanosecond-resolution timing
class CycleCounter {
private:
    static constexpr uint32_t DWT_CYCCNT = 0xE0001004;
    static constexpr uint32_t DWT_CONTROL = 0xE0001000;
    static constexpr uint32_t DEMCR = 0xE000EDFC;
    
public:
    static void enable() {
        // Enable trace system
        CoreDebug->DEMCR |= CoreDebug_DEMCR_TRCENA_Msk;
        
        // Enable cycle counter
        DWT->CYCCNT = 0;
        DWT->CTRL |= DWT_CTRL_CYCCNTENA_Msk;
    }
    
    static uint32_t read() {
        return DWT->CYCCNT;
    }
    
    static uint32_t cycles_to_us(uint32_t cycles) {
        return cycles / (SystemCoreClock / 1000000);
    }
    
    static uint32_t us_to_cycles(uint32_t microseconds) {
        return microseconds * (SystemCoreClock / 1000000);
    }
};
```

### UART Interrupt-Based Timestamping with DMA

The `TimestampedUART` class implements hardware timestamping for UART reception using DMA. The `init()` method configures DMA in circular mode with interrupt on transfer complete (`DMA_SxCR_TCIE`). Each received byte is stored in `rx_buffer` along with a `timestamp_cycles` captured via `CycleCounter::read()`. The `dma_isr()` handler updates the head pointer when DMA transfer completes. The `get_reception_time_us()` function calculates the exact reception time by comparing the captured cycle count with the current cycle count, handling 32-bit overflow. This provides microsecond-accurate timestamps for jitter correction.
