# Chapter: Newton-Raphson Iteration

## Introduction

The Newton-Raphson iteration forms the computational core of Ngspice's nonlinear circuit solver, implemented primarily in `niniter.c` and related numerical infrastructure files. This algorithm solves the system of nonlinear algebraic-differential equations generated by modified nodal analysis, transforming the circuit simulation problem into a sequence of linear systems. At each DC operating point or transient analysis time step, the implementation iteratively linearizes device equations around the current voltage solution, constructs and solves a Jacobian system, and updates node voltages until convergence criteria are satisfied. The code combines mathematical rigor with practical numerical stabilization techniques—including damping, GMIN stepping, and singularity handling—to achieve robust convergence across diverse circuit topologies, from well-behaved digital logic to challenging analog circuits with high gain and feedback.

## Mathematical Formulation

The Newton-Raphson iteration in SPICE solves the system of nonlinear algebraic-differential equations arising from modified nodal analysis:

\[
\mathbf{F}(\mathbf{x}, \dot{\mathbf{x}}, t) = \mathbf{0}
\]

where \(\mathbf{x} \in \mathbb{R}^n\) is the vector of node voltages (and branch currents for voltage sources), \(\dot{\mathbf{x}}\) represents time derivatives, and \(\mathbf{F}\) encapsulates Kirchhoff's Current Law at each node combined with device constitutive relations.

For DC analysis or at a fixed time point in transient analysis (using implicit integration), the problem reduces to solving \(\mathbf{F}(\mathbf{x}) = \mathbf{0}\). Given an initial guess \(\mathbf{x}^{(0)}\), the algorithm iteratively linearizes the system. At iteration \(k\):

\[
\mathbf{J}^{(k)} \Delta\mathbf{x}^{(k)} = -\mathbf{F}(\mathbf{x}^{(k)})
\]
\[
\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} + \lambda^{(k)} \Delta\mathbf{x}^{(k)}
\]

Here, \(\mathbf{J}^{(k)} = \frac{\partial \mathbf{F}}{\partial \mathbf{x}} \big|_{\mathbf{x}^{(k)}}\) is the Jacobian matrix, \(\Delta\mathbf{x}^{(k)}\) is the Newton correction step, and \(\lambda^{(k)} \in (0, 1]\) is a damping factor for convergence control.

The Jacobian is constructed from the linearized stamps of all circuit elements. For nonlinear devices, this requires computing the derivative of their branch current with respect to terminal voltages. For example:
*   For a nonlinear resistor defined by \(I = f(V)\): \(g = \frac{df}{dV}\).
*   For a diode with \(I_D = I_S(e^{V_D/(nV_T)} - 1)\): \(g_D = \frac{I_S}{nV_T}e^{V_D/(nV_T)} \approx \frac{I_D + I_S}{nV_T}\).
*   For a MOSFET in saturation (\(I_D = \frac{\mu C_{ox}}{2}\frac{W}{L}(V_{GS} - V_{th})^2(1 + \lambda V_{DS})\)): \(g_m = \frac{\partial I_D}{\partial V_{GS}} = \mu C_{ox}\frac{W}{L}(V_{GS} - V_{th})(1 + \lambda V_{DS})\) and \(g_d = \frac{\partial I_D}{\partial V_{DS}} = \frac{\mu C_{ox}}{2}\frac{W}{L}(V_{GS} - V_{th})^2\lambda\).

The right-hand side vector \(\mathbf{F}(\mathbf{x}^{(k)})\) represents the net current flowing into each node, computed from the device equations evaluated at \(\mathbf{x}^{(k)}\). The linear system \(\mathbf{J}\Delta\mathbf{x} = -\mathbf{F}\) is solved using sparse LU factorization. To handle numerically singular Jacobians, SPICE may apply a small diagonal perturbation before reattempting the factorization.

The update includes a damping mechanism: \(\lambda^{(k)}\) is reduced, often to a minimum of 0.1, if the raw Newton step \(\Delta\mathbf{x}^{(k)}\) exceeds a maximum allowed voltage change (e.g., 10V for MOS devices) to prevent overshoot and divergence.

## Convergence Analysis

Convergence of the Newton-Raphson iteration in SPICE is determined by checking three simultaneous criteria for every node \(i\) in the circuit after each update \(\mathbf{x}^{(k)} \to \mathbf{x}^{(k+1)}\). Let \(F_i\) be the \(i\)-th component of the residual vector \(\mathbf{F}\).

1.  **Absolute Current Tolerance**: \(|F_i(\mathbf{x}^{(k+1)})| < \epsilon_{abs}\). The default \(\epsilon_{abs} = 10^{-12}\) A ensures the net current imbalance at each node is negligible.
2.  **Relative Current Tolerance**: \(\frac{|F_i(\mathbf{x}^{(k+1)})|}{\max(|F_i(\mathbf{x}^{(k)})|, |F_i(\mathbf{x}^{(k+1)})|)} < \epsilon_{rel}\). The denominator uses the maximum of the current and previous residuals to avoid division by zero. The default \(\epsilon_{rel} = 10^{-3}\) ensures the residual is decreasing sufficiently.
3.  **Absolute Voltage Change Tolerance**: \(|x_i^{(k+1)} - x_i^{(k)}| < \epsilon_{volt}\). The default \(\epsilon_{volt} = 50 \times 10^{-6}\) V ensures the solution is no longer changing significantly.

The iteration is declared converged only when **all three conditions are satisfied for all nodes**. This stringent requirement guarantees a solution that is both electrically accurate (per KCL) and numerically stable.

If the standard Newton iteration fails to converge within a set limit (e.g., 100 iterations), SPICE employs auxiliary algorithms to extend the region of convergence:
*   **Damping**: Reducing \(\lambda\) tempers the Newton step, trading convergence rate for stability.
*   **GMIN Stepping**: A small conductance \(G_{min}\) is added from every node to ground. The simulation starts with a large \(G_{min}\) (e.g., \(10^{-3}\) S) where convergence is more likely, then iteratively reduces it toward its nominal value (e.g., \(10^{-12}\) S), using the previous solution as the initial guess for the next step. This technique helps overcome issues with high-impedance nodes and poorly conditioned Jacobians.
*   **Source Stepping**: Independent source values are scaled from zero (or a small fraction) to their full value. Convergence is sought at each intermediate source level, progressively homotoping the circuit to its final operating point.

The algorithm exhibits local quadratic convergence when near a solution: \(\|\mathbf{x}^{(k+1)} - \mathbf{x}^*\| = O(\|\mathbf{x}^{(k)} - \mathbf{x}^*\|^2)\), provided the Jacobian is non-singular. However, its global convergence properties depend entirely on the initial guess and the circuit's topology. The aforementioned stepping techniques are critical for achieving convergence from a poor initial guess (often the default of all zeros). Failure modes include oscillation (sign-alternating steps of similar magnitude), monotonic divergence, and numerical singularity of the Jacobian, each triggering specific fallback strategies in the implementation.

## C Implementation

### Core Data Structures

The algorithm's state is maintained in two primary structures:

```c
typedef struct {
    double *rhs;           /* Right-hand side vector F(x) */
    double *sol;           /* Solution vector x */
    double *oldsol;        /* Previous solution x^(k-1) */
    double *saveRHS;       /* Saved RHS for convergence checking */
    int matrix_size;       /* Dimension of the system */
    int iteration;         /* Current Newton iteration count */
    int max_iterations;    /* Maximum allowed iterations (e.g., 100) */
    double abstol;         /* Absolute tolerance */
    double reltol;         /* Relative tolerance */
    double vntol;          /* Voltage tolerance */
    double damping_factor; /* λ for step limiting */
    int converged;         /* Convergence flag */
} NIIterationState;
```

This structure encapsulates the mathematical variables: `rhs` corresponds to \(\mathbf{F}(\mathbf{x}^{(k)})\), `sol` to \(\mathbf{x}^{(k)}\), and `oldsol` to \(\mathbf{x}^{(k-1)}\). The tolerance fields store \(\epsilon_{abs}\), \(\epsilon_{rel}\), and \(\epsilon_{volt}\) with defaults of \(10^{-12}\), \(10^{-3}\), and \(50 \times 10^{-6}\) respectively.

```c
typedef struct {
    double *matrix;        /* Jacobian matrix (sparse format) */
    int *col_ptr;          /* Column pointers for sparse storage */
    int *row_idx;          /* Row indices for sparse storage */
    int nnz;               /* Number of non-zero elements */
    int factored;          /* LU factorization status flag */
    double *pivot;         /* Pivot elements for LU */
    int *perm;             /* Permutation vector */
} JacobianSystem;
```

This structure stores the Jacobian matrix \(\mathbf{J}^{(k)}\) in a sparse format (typically compressed sparse column) along with its LU factorization components, enabling efficient solution of the linear system \(\mathbf{J}\Delta\mathbf{x} = -\mathbf{F}\).

### Main Iteration Loop

The primary Newton-Raphson iteration is implemented in a function structured as follows:

```c
int newton_iteration(NIIterationState *state, JacobianSystem *jacobian) {
    int iter = 0;
    state->converged = 0;
    
    /* Save initial RHS for relative tolerance calculation */
    memcpy(state->saveRHS, state->rhs, state->matrix_size * sizeof(double));
    
    while (iter < state->max_iterations && !state->converged) {
        /* 1. Evaluate device equations at current solution */
        evaluate_device_equations(state->sol, state->rhs);
        
        /* 2. Build Jacobian matrix */
        build_jacobian(state->sol, jacobian);
        
        /* 3. Solve linear system: JΔx = -F */
        solve_linear_system(jacobian, state->rhs, delta_x);
        
        /* 4. Update solution with damping */
        update_solution(state, delta_x);
        
        /* 5. Check convergence */
        state->converged = check_convergence(state, iter);
        
        iter++;
        state->iteration = iter;
    }
    
    return state->converged ? CONVERGED : MAX_ITERATIONS_EXCEEDED;
}
```

This loop directly implements the mathematical iteration: \(\mathbf{J}^{(k)}\Delta\mathbf{x}^{(k)} = -\mathbf{F}(\mathbf{x}^{(k)})\) followed by \(\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} + \lambda^{(k)}\Delta\mathbf{x}^{(k)}\). The `evaluate_device_equations()` function computes \(\mathbf{F}(\mathbf{x}^{(k)})\) by calling device model callbacks, while `build_jacobian()` constructs \(\frac{\partial\mathbf{F}}{\partial\mathbf{x}}\big|_{\mathbf{x}^{(k)}}\) from device derivatives.

### Convergence Checking Implementation

The three-part convergence criterion is implemented as:

```c
int check_convergence(NIIterationState *state, int iteration) {
    int i;
    int all_converged = 1;
    double abs_err, rel_err, volt_change;
    
    /* Evaluate new RHS at updated solution */
    evaluate_device_equations(state->sol, state->rhs);
    
    for (i = 0; i < state->matrix_size; i++) {
        abs_err = fabs(state->rhs[i]);
        
        /* Relative error uses max(|F_i^k|, |F_i^{k-1}|) to avoid division by zero */
        double denom = fmax(fabs(state->saveRHS[i]), fabs(state->rhs[i]));
        rel_err = (denom > 0) ? abs_err / denom : 0.0;
        
        volt_change = fabs(state->sol[i] - state->oldsol[i]);
        
        /* Three-way convergence check */
        if (abs_err > state->abstol && 
            rel_err > state->reltol && 
            volt_change > state->vntol) {
            all_converged = 0;
            break;
        }
    }
    
    /* Save current RHS for next iteration's relative check */
    memcpy(state->saveRHS, state->rhs, state->matrix_size * sizeof(double));
    
    return all_converged;
}
```

This code checks \(|F_i(x^{(k+1)})| < \epsilon_{abs}\), \(\frac{|F_i(x^{(k+1)})|}{\max(|F_i(x^{(k)})|, |F_i(x^{(k+1)})|)} < \epsilon_{rel}\), and \(|x_i^{(k+1)} - x_i^{(k)}| < \epsilon_{volt}\) for all nodes \(i\). The relative tolerance calculation uses the maximum of current and previous residuals to avoid division by near-zero values.

### Damped Solution Update

The update step with damping factor \(\lambda\) is implemented as:

```c
void update_solution(NIIterationState *state, double *delta_x) {
    int i;
    double max_step = 0.0;
    double damping = state->damping_factor;
    
    /* Save old solution for convergence checking */
    memcpy(state->oldsol, state->sol, state->matrix_size * sizeof(double));
    
    /* Calculate maximum step size for potential step limiting */
    for (i = 0; i < state->matrix_size; i++) {
        double step = fabs(delta_x[i]);
        if (step > max_step) max_step = step;
    }
    
    /* Apply step limiting if needed (typical limit: 50V for bipolar, 10V for MOS) */
    if (max_step > MAX_ALLOWED_STEP) {
        damping *= MAX_ALLOWED_STEP / max_step;
    }
    
    /* Apply damped Newton update */
    for (i = 0; i < state->matrix_size; i++) {
        state->sol[i] += damping * delta_x[i];
    }
}
```

This implements \(\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} + \lambda^{(k)}\Delta\mathbf{x}^{(k)}\) with additional step limiting. If any component of \(\Delta\mathbf{x}^{(k)}\) exceeds `MAX_ALLOWED_STEP` (typically 10V for MOS devices), \(\lambda^{(k)}\) is reduced proportionally to prevent numerical instability.

### Linear System Solution with Singularity Handling

The solution of \(\mathbf{J}\Delta\mathbf{x} = -\mathbf{F}\) includes handling of numerically singular matrices:

```c
int solve_linear_system(JacobianSystem *jac, double *rhs, double *solution) {
    int status;
    
    /* Factorize Jacobian if not already factored */
    if (!jac->factored) {
        status = lu_factorize(jac->matrix, jac->pivot, jac->perm, jac->nnz);
        if (status == SINGULAR_MATRIX) {
            /* Apply matrix perturbation or alternative strategy */
            perturb_diagonal(jac->matrix, jac->nnz, PERTURBATION_VALUE);
            status = lu_factorize(jac->matrix, jac->pivot, jac->perm, jac->nnz);
        }
        jac->factored = 1;
    }
    
    /* Solve using factored matrix */
    lu_solve(jac->matrix, jac->pivot, jac->perm, rhs, solution);
    
    return status;
}
```

When LU factorization detects a singular or near-singular Jacobian, the code perturbs the diagonal elements before reattempting factorization, ensuring the linear system can be solved.

### Convergence Acceleration: GMIN Stepping

For difficult convergence cases, Ngspice implements GMIN stepping:

```c
int gmin_stepping(NIIterationState *state, JacobianSystem *jac) {
    double gmin_orig = Gmin_DEFAULT;  /* e.g., 1e-12 */
    double gmin_current;
    
    for (gmin_current = GMIN_START; gmin_current >= gmin_orig; gmin_current /= GMIN_FACTOR) {
        /* Add conductance to ground at each node */
        add_gmin_to_jacobian(jac, gmin_current);
        
        if (newton_iteration(state, jac) == CONVERGED) {
            /* Success - continue to next lower gmin */
            if (gmin_current == gmin_orig) return CONVERGED;
        } else {
            /* Failure - backtrack to previous successful gmin */
            return RESTART_WITH_HIGHER_GMIN;
        }
    }
    
    return FAILED_TO_CONVERGE;
}
```

This algorithm starts with a large conductance \(G_{min} = 10^{-3}\) S from each node to ground, making the Jacobian better conditioned. It then iteratively reduces \(G_{min}\) by factors of 10 until reaching the nominal value of \(10^{-12}\) S, using the solution from each stage as the initial guess for the next.
