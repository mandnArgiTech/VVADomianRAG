## Mathematical Formulation

### 1. Continuous-Discrete Variable Bridging Mathematics

XSPICE bridges continuous-time analog variables with discrete-time event variables through formal mathematical transformations that maintain consistency with SPICE's Modified Nodal Analysis (MNA) framework.

#### 1.1 Analog-to-Digital Conversion (ADC) Model Formulation

The ADC model implements a threshold-based mapping from continuous voltage to discrete digital states:

```
DigitalOutput[n] = 
    ‚éß 1 if V_analog ‚Č• V_thresh_high[n]
    ‚é® 0 if V_analog ‚Č§ V_thresh_low[n]

```

This creates a hysteresis band to prevent chatter:
```
V_thresh_high[n] = V_nominal_high + őĒV_hysteresis/2
V_thresh_low[n] = V_nominal_low - őĒV_hysteresis/2
```

For multi-bit ADCs with N bits, the quantization function is:
```

```
where ‚Ćä¬∑‚Ćč denotes floor operation, and each bit `b_k` is extracted as:
```
b_k = (DigitalCode >> k) & 1 for k = 0,...,N-1
```

#### 1.2 Digital-to-Analog Conversion (DAC) Model Formulation

The DAC implements the inverse transformation from discrete digital states to continuous voltage:

```
V_analog = V_low + (V_high - V_low) √ó (‚ąĎ_{i=0}^{N-1} bit[i]¬∑2^i)/(2^N - 1)
```

For weighted resistor DACs, the output impedance contributes to the MNA matrix:
```

```
where `R_ladder` is the ladder network impedance and `R_i` are weighted resistor values.

where `f(V_DD, T)` models voltage and temperature dependence.


- `J_dd ‚ąą ‚ĄĚ^{n_d√ón_d}`: Digital-digital block (logic fanout, typically 0 or 1)


### 2. Event Queue Dynamics and Time Management

### 3. Code Model Jacobian Computation

#### 3.1 Nonlinear Device Contribution to MNA


### 3. Event Queue Stability Analysis

#### 3.1 Event Scheduling Consistency

**Timestamp Monotonicity:**
All events must satisfy:
```

```
If an event is scheduled with t < t_current, it's processed immediately.

**Event Causality:**
Events cannot affect their own cause. Formally, for event chain e‚āĀ ‚Üí e‚āā ‚Üí ... ‚Üí e‚āô:

Circular causality is detected and broken by:
1. Identifying strongly connected components in event dependency graph
2. Sorting events within each SCC by priority
3. Processing events in priority order at the same timestamp

#### 3.2 Memory Management for Event Queues

- ŌÉ_events = standard deviation of event count

```c
            new_state = 1;  /* V_analog ‚Č• V_thresh_high */
        } else if (v <= iface->threshold_low) {
            new_state = 0;  /* V_analog ‚Č§ V_thresh_low */
        } else {
            /* Hysteresis region */
            if (iface->digital_state == 1 && v > iface->threshold_low + iface->hysteresis) {
                new_state = 1;
            } else if (iface->digital_state == 0 && v < iface->threshold_high - iface->hysteresis) {
                new_state = 0;
            } else {
                new_state = iface->digital_state;  /* Maintain previous state */
            }
        }
        
        /* Mathematical: Schedule event if state changed */
        if (new_state != iface->digital_state) {
            /* Create event for state transition */
            Event *ev = create_digital_event(iface->node_index, new_state, ckt->CKTtime);

        }
    }
```

```c
double cm_digital_to_analog(int *digital_bits, int num_bits, 
                           double v_low, double v_high) {
    /* Mathematical: Compute weighted sum of bits */
    int sum = 0;
    for (int i = 0; i < num_bits; i++) {
        if (digital_bits[i]) {
            sum += (1 << i);  /* bit[i]¬∑2^i */
        }
    }
    
    /* Mathematical: Scale to voltage range */
    double scale = (double)sum / ((1 << num_bits) - 1);  /* (‚ąĎ bit[i]¬∑2^i)/(2^N - 1) */
    return v_low + (v_high - v_low) * scale;
}
```

```c
/* Mathematical Mapping: Inserts event in time-sorted order */
int schedule_event(EventQueue *queue, Event *ev) {
    /* Mathematical: Find insertion position to maintain t_next = min_{e‚ąąEventQueue} (e.timestamp) */
    Event *prev = NULL;
    Event *curr = queue->head;
    
    /* Traverse to find correct position */
    while (curr && curr->timestamp <= ev->timestamp) {
        prev = curr;
        curr = curr->next;
    }
    
    /* Insert event */
    ev->next = curr;
    if (prev) {
        prev->next = ev;
    } else {
        queue->head = ev;  /* New earliest event */
    }
    
    if (!curr) {
        queue->tail = ev;  /* New latest event */
    }
    
    queue->count++;
    
    /* Mathematical: Check for Zeno behavior prevention */
    if (prev && ev->timestamp - prev->timestamp < MIN_EVENT_SPACING) {
        /* Merge events if too close in time */
        return merge_events(prev, ev);
    }
    
    return OK;
}

/* Mathematical Mapping: Processes events up to current time */
int cm_process_events(CKTcircuit *ckt) {
    CMManager *mgr = ckt->CKTcmManager;
    EventQueue *queue = mgr->event_queue;
    double current_time = ckt->CKTtime;
    
    /* Mathematical: Process all events with timestamp ‚Č§ current_time */
    while (queue->head && queue->head->timestamp <= current_time) {
        Event *ev = queue->head;
        queue->head = ev->next;
        
        /* Apply event to circuit */
        switch (ev->event_type) {
            case EVENT_DIGITAL:
                /* Mathematical: Update digital node value */
                ckt->CKTdigitalStates[ev->node_index] = ev->value.digital_value;
                break;
                
            case EVENT_ANALOG:
                /* Mathematical: Update analog node voltage */
                ckt->CKTrhs[ev->node_index] = ev->value.analog_value;
                break;
                
            case EVENT_STATE:
                /* Mathematical: Update code model internal state */
                cm_apply_state_event(ckt, ev);
                break;
        }
        
        free_event(ev);
        queue->count--;
    }
    
    /* Mathematical: Update last process time */
    queue->last_process_time = current_time;
    
    return OK;
}
```

**Mathematical Mapping**: The event queue implements the time-advance function `t_next = min_{e‚ąąEventQueue} (e.timestamp)` by maintaining events in time-sorted order. The Zeno behavior prevention condition `őĒt_min = max(őĶ_time, 10¬∑t_step/tol)` is enforced through event merging.
