## Business Context

We now have **176 ngspice domain doc chapters** (~4.7 MB) in `Studio-Portable-RAG/DomainDocs/ngspice/`, generated by `crewai/ngspice_book_factory.py`. These chapters cover every device model (BSIM1–4v7, MOS1–9, BJT, diode, resistor, capacitor, inductor, transmission lines, switches, JFET, HFET, MESFET, VBIC, CIDER), every analysis type (DC, AC, transient, pole-zero, distortion, sensitivity, PSS), every numerical subsystem (Newton-Raphson, integration methods, sparse matrix, device limiters, GMIN/source stepping), and every parser/XSPICE subsystem.

**The problem:** These docs are markdown with a very specific structure — each chapter has:
1. A `**Source files:**` header listing the exact ngspice C source files the chapter was generated from
2. LaTeX math blocks (`\[...\]`) with device equations
3. C code blocks showing the actual ngspice implementation
4. Sections following a consistent pattern: Introduction → Mathematical Formulation → Convergence Analysis → C Implementation → Constants

**The opportunity:** If we ingest these correctly, a single RAG query for "diode junction voltage limiting" should return:
- The **domain doc chunk** from Chapter_05 explaining DEVpnjlim math + C code
- The **C source chunk** of the actual `DEVpnjlim` function (from ngspice code ingest)
- The **NodalAI Python chunk** of `_limit_junction_voltage` (from ecad code ingest)

All three in one retrieval — theory + reference implementation + our implementation. This is the triple-context that makes debugging convergence issues tractable.

To achieve this, we need:
1. **Expanded concept registry** that covers the vocabulary in these domain docs
2. **Source-file cross-referencing** — parse the `**Source files:**` header and store as metadata so the MCP can link domain doc chunks to code chunks from the same file
3. **Math-aware chunking** that doesn't split LaTeX blocks mid-equation

### Step 2: Add `source_c_files` extraction to `chunk_markdown_domain`

In `ingest.py`, inside `chunk_markdown_domain()`, before chunking begins, parse the source files header:

```python
def _extract_source_c_files(text: str) -> str:
    """Extract C source file basenames from **Source files:** header in domain docs."""
    m = re.search(r'\*\*Source files:\*\*\s*\n((?:\s*-\s*.+\n)*)', text)
    if not m:
        return ""
    lines = m.group(1).strip().split('\n')
    basenames = []
    for line in lines:
        line = line.strip().lstrip('- ').strip('`').strip()
        if line:
            basenames.append(Path(line).name)
    return ", ".join(basenames)
```

Pass the result into the chunk metadata dict.

### Step 3: Add `source_c_files` and `chapter_number` to METADATA_KEYS

Add `"source_c_files"` and `"chapter_number"` to the `METADATA_KEYS` list.

### Step 4: Math block protection in paragraph splitting

In `_split_paragraphs()` (used by `chunk_markdown_domain`), before splitting on `\n\n`, replace math blocks with placeholders:

```python
def _protect_math_blocks(text: str) -> Tuple[str, Dict[str, str]]:
    """Replace \\[...\\] and $$...$$ with placeholders to prevent mid-equation splits."""
    vault = {}
    for i, m in enumerate(re.finditer(r'\\\[.*?\\\]|\$\$.*?\$\$', text, re.DOTALL)):
        key = f"__MATH_BLOCK_{i}__"
        vault[key] = m.group(0)
        text = text.replace(m.group(0), key, 1)
    return text, vault
```

After splitting, restore math blocks in each chunk.

def _chapter_meta_from_filename(filename: str) -> Dict[str, str]:
    m = re.match(r'Chapter_(\d+)_(.*)', Path(filename).stem)

        'Device': 'CORE', 'MNA': 'CORE', 'Simulation': 'CORE',
        'Core': 'CORE', 'Complex': 'CORE', 'Derivative': 'CORE',
