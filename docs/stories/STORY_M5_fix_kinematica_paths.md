## Implementation

### Step 1 — Add a path-validator helper

Create `crewai/scripts/validate_oracle_paths.py`:

```python
"""Validate that every file path in an oracle JSON resolves under source_root."""
from __future__ import annotations
import json, sys
from pathlib import Path

def validate(oracle_path: Path, source_root: Path) -> list[tuple[str, str]]:
    """Return a list of (chapter_key, missing_file) tuples."""
    data = json.loads(oracle_path.read_text())
    missing = []
    for ch_key, ch in data.items():
        for f in ch.get("files", []):
            full = source_root / f
            if not full.exists():
                missing.append((ch_key, f))
    return missing

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: validate_oracle_paths.py <oracle.json> <source_root>")
        sys.exit(2)
    missing = validate(Path(sys.argv[1]), Path(sys.argv[2]))
    if missing:
        print(f"MISSING ({len(missing)}):")
        for ch, f in missing:
            print(f"  [{ch}]  {f}")
        sys.exit(1)
    print("All paths resolve.")
```

### Step 2 — Apply path-prefix rules to `oracle_kinematica.json`

Write a one-shot migration script `crewai/scripts/fix_kinematica_paths.py` that reads the oracle, applies the rules in the table above, and writes the corrected JSON back. Validate against your local ArduPilot checkout before committing.

### Step 3 — Wire validation into the book factory CLI

In `crewai/book_factory/cli.py`, before iterating chapters, run the validator and **fail fast** with a clear error listing every missing file. This prevents silent hallucinated chapters in future runs.
